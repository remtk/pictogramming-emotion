# ピクトグラミング＋感情表現（Pictogramming Emotion Edition）

人型ピクトグラムを命令文で動かしてプログラミングを学ぶ「ピクトグラミング」
（伊藤一成, 2018, 情報処理学会論文誌 教育とコンピュータ Vol.4 No.2）の設計指針
（実装指針1〜3、ピクトアニメーション／ピクトグラフィックスの2系統の命令）を参考に、
新規拡張として **感情表現（`EMOTION`命令）** を追加した非公式の学習用実装です。

公式の Pictogramming（<https://pictogramming.org/editor/>）とは独立したコードベースです。
node + ブラウザのみで動作し、外部ライブラリへの依存はありません。

## 新規性: 感情表現コマンド

元の論文の人型ピクトグラムは「動作の模倣による同調的学習」を主題としていますが、
喜怒哀楽そのものを表す手段は用意されていませんでした。本実装では新たに

```
EMOTION <感情名> [<秒>]
```

という命令を追加し、以下の5種類の感情について

- 表情アイコン（顔の上に簡易的な目・口を表示）
- 色のオーラ（頭部周農の色付きリング、CSSアニメーションで脈動）
- 標準ポーズへの遷移（指定秒数をかけて腕などの角度を補間）

を同時に変化させることで、人型ピクトグラムに感情を表現できるようにしました。

| 感情名（日本語） | キー | 色 |
|---|---|---|
| よろこび | `JOY` | 琥珀色 |
| 悲しみ | `SAD` | 鈍い青 |
| 怒り | `ANGRY` | 朱赤 |
| 驚き | `SURPRISE` | 紫 |
| 普通 | `NORMAL` | 無彩色（標準） |

英語表記・漢字表記・ひらがな表記のいずれでも指定できます（例: `EMOTION JOY` / `EMOTION 喜び` / `EMOTION よろこび`）。

## 起動方法

```bash
npm start
# http://localhost:3000 をブラウザで開く
```

依存パッケージのインストールは不要です（Node標準モジュールのみで静的サーバを実装しています）。

## テスト

```bash
npm test
```

パーサー・式評価・部位名解決・感情名解決についてのユニットテストを `test/` に用意しています。

## サポートしている命令

### ピクトアニメーション命令
- `R 部位 角度` … 指定部位を瞬時に回転
- `RW 部位 角度 秒` … 指定秒数をかけて回転（アニメーション）
- `M x y` … 体全体を瞬時に平行移動
- `MW x y 秒` … 指定秒数をかけて平行移動

部位名は `BODY / LUA / LLA / RUA / RLA / LUL / LLL / RUL / RLL` の英語表記の他、
「体」「左上腕」など漢字表記、「からだ」「ひだりじょうわん」などひらがな表記にも対応しています。

### ピクトグラフィックス命令
- `PEN DOWN` / `PEN UP` … ペンの上げ下げ（移動軌跡の描画開始/終了）
- `PEN COLOR "#色"` … 描画色の変更

### 共通命令
- `REPEAT n ... END` … 繰返し
- `IF [式] ... END` … 条件分岐（JavaScriptの算術・比較演算子がそのまま使用可能）
- `LET 変数 値` … 変数定義
- `WAIT 秒` … 待機
- `SP "文字列"` … 吹き出しでの発話表示

### 感情表現命令（新規拡張）
- `EMOTION 感情名 [秒]` … 上記表の感情へ遷移する

## 既知の制約・今後の拡張案

- 側面ピクトグラムは未対応（正面のみ）。論文中でも判別性の課題が指摘されており、今後の検討課題としています。
- ピクトグラフィックスの線描画は胴体の下に重なって見えにくい場合があります。描画レイヤーの最前面化は今後の改善点です。
- 感情の強度（弱い喜び／強い喜びなど）の段階表現には対応していません。
- ブロックプログラミング（Blockly等）への対応は行っていません。テキスト入力のみです。
